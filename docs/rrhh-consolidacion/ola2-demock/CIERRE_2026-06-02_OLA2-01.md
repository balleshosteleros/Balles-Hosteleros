## Aprendizaje operativo (importante)

En **dev (Turbopack HMR)** las server actions fallan intermitentemente con `Failed to find Server Action … older/newer deployment` tras recompilar (los IDs de action cambian al hot-reload). Provoca falsos **"0 empleados" / "Cargando…" / "Failed to fetch"** en pantallas que llaman actions al montar (afecta también a páginas no tocadas como `/rrhh/empleados` y la carga de permisos). **No es bug de producto, no afecta a producción** (IDs estables), y desaparece al asentarse el server. En smokes: esperar a que compilen las rutas (waits ≥10s en el primer hit) o usar el server ya asentado; si aparece "0", refrescar.
