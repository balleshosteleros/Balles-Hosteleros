## Reglas de aplicación

1. **Si la palabra corta cambia el matiz técnico, NO sustituyas**. Ej: "verificar" en contexto de testing puede ser específico, no siempre cambiar a "revisar".
2. **No sustituyas dentro de bloques de código ni en nombres propios**.
3. **Una sustitución por aparición**, no inventes contextos.
4. **Si la sustitución hace que la frase suene rara o ambigua, mantén la original**.
